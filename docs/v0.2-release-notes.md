# TLS Audit v0.2 — Release Notes

Date: 2026-05-29
Status: v0.2 baseline with v0.2.1 maintenance update

## Included in v0.2

1. Public HTTPS/TLS scan report with:
   - certificate checks;
   - TLS protocol checks;
   - cipher suite checks;
   - HSTS/header checks;
   - vulnerability signals;
   - explanation-first recommendations.

2. Scoring and grading model with explicit downgrade reasons.

3. Monitoring core:
   - domain subscriptions (free/pro);
   - email confirmation;
   - scheduler + worker queue execution;
   - snapshots, events, and diff model.

4. Pro flow controls:
   - ownership verification flow;
   - delivery guard when ownership is not verified;
   - anti-duplicate and anti-storm logic for report/alert delivery.

5. Digest email productization:
   - “Security status digest” style;
   - concise, softer language;
   - positive checks block;
   - link to full report and monitoring panel.

6. API and operational hardening:
   - monitoring trust-zone/admin access protection;
   - SSRF-oriented target validation in scan flow;
   - run-now/scheduled archive FK path fixed.

7. Frontend polish:
   - report card stretch fix;
   - monitoring/report UI cleanup.

## Not included in v0.2

1. Payment processing and production billing.
2. Full user account system with dashboard auth.
3. Multi-channel alerts (beyond email).
4. Advanced external integrations/webhooks as product feature set.
5. Full comparative rating parity with SSL Labs.

## Upgrade/Deployment Notes

1. Use `main` branch baseline commit for v0.2 pause.
2. Deploy via Docker Compose (`api`, `worker`, `scheduler`, `redis`, `postgres`).
3. Keep SMTP configured for monitoring digests.
4. Keep DB backups before major post-v0.2 changes.

## v0.2.1 Maintenance Update — 2026-07-13

1. Closed confirmation-link disclosure and removed public Pro demo activation/status APIs.
2. Added expiring owner links and mandatory production token secret.
3. Pinned ownership and scan connections to validated public IP addresses.
4. Prevented duplicate subscription scheduling and disabled orphaned monitored domains.
5. Added lifetime scan counter, stale-subscription cleanup, and real dependency readiness checks.
6. Hardened containers with non-root users, read-only filesystems, dropped capabilities, and no-new-privileges.
7. Updated FastAPI/Starlette and container pip; the final API image has no known vulnerable packages.
