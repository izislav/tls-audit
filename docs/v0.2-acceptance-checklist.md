# TLS Audit v0.2 — Acceptance Checklist

Date: 2026-05-29  
Scope: functional and operational acceptance for v0.2 pause baseline

## 1) Public Scan Flow

- [x] `POST /api/check` creates scan job.
- [x] `GET /api/check/{id}` returns progressing status until `done`/`error`.
- [x] `GET /api/report/{id}` returns structured report JSON.
- [x] Report page renders grade, reasons, findings, and recommendations.

## 2) Monitoring Core

- [x] Monitoring subscription can be created for `free` plan.
- [x] Monitoring subscription can be created for `pro` plan.
- [x] Email confirmation flow works (confirm/unsubscribe links).
- [x] Scheduler picks due subscriptions and enqueues scan jobs.
- [x] Worker saves scan snapshots and comparison baseline.
- [x] Run-now action triggers immediate monitoring scan.

## 3) Pro Ownership Flow

- [x] Pro subscription exposes ownership challenge flow.
- [x] Verification state is stored and shown in monitor-status panel.
- [x] Pro delivery is blocked when ownership is not verified.
- [x] Pro delivery is enabled after successful ownership verification.

## 4) Digest and Alert Delivery

- [x] Weekly security digest emails are sent for confirmed subscriptions.
- [x] Duplicate report guard blocks repeated delivery for same scan context.
- [x] Cooldown logic reduces alert/report storm on repeated scans.
- [x] Digest includes concise status + full report link + monitoring panel link.

## 5) Security and Access Control

- [x] Monitoring internal/admin surface is restricted by trust-zone checks.
- [x] Public scan input rejects private/internal targets (SSRF guardrail path).
- [x] Monitoring API behavior is separated from public scan surface.

## 6) Data/Schema and Runtime

- [x] PostgreSQL schema supports scans, subscriptions, snapshots, events, archives.
- [x] Redis queue is used for async job execution.
- [x] Docker Compose stack runs `api`, `worker`, `scheduler`, `redis`, `postgres`.
- [x] Archive FK path for scheduled/run-now save is fixed.

## 7) UI/UX Baseline

- [x] Main page scan controls are aligned and responsive.
- [x] Monitoring/pro sections are present and functional.
- [x] Report page no longer stretches oversized recommendation area.
- [x] Monitoring status panel is accessible by secure token link.

## 8) Operational Readiness

- [x] SMTP credentials configured and test send path validated.
- [x] Canonical domain is `https://tlsaudit.ru`.
- [x] Project docs updated with v0.2 status and release notes.

## 9) Out-of-Scope (Explicitly Deferred)

- [ ] Payment integration (gateway + billing lifecycle).
- [ ] Full user accounts and authenticated dashboard.
- [ ] Multi-channel notifications (email + webhook + messenger set).
- [ ] Full SSL Labs rating parity and extended methodology publication set.

Result: v0.2 accepted as stable baseline for pause and controlled next-phase work.
