# TLS Audit Growth Plan

Этот план фиксирует этапы после запуска MVP на `tlsaudit.ru`: стабильность,
доверие, продвижение и продуктовые функции. Выполняем последовательно, чтобы не
раздувать сервис раньше времени.

## 1. Публичное доверие и юридическая аккуратность

Статус: started.

- добавить страницы: политика обработки персональных данных, условия
  использования, cookies, безопасность/допустимое использование;
- добавить ссылки на документы в интерфейс;
- добавить рабочий контактный email для abuse/security/privacy;
- перед публичным продвижением финально проверить тексты юристом;
- понять, требуется ли уведомление Роскомнадзора как оператору персональных
  данных, если сохраняются IP-адреса, логи и отчеты.
- DNS/MX для `admin@tlsaudit.ru` еще нужно настроить: MX/TXT сейчас не
  опубликованы.

## 2. Базовая аналитика без лишних cookies

Статус: started.

- на первом этапе использовать статистику Nginx/API логов: визиты, проверки,
  ошибки, популярные страницы входа;
- не подключать рекламные трекеры и вебвизор до появления политики согласия;
- если подключается Яндекс Метрика, сначала обновить cookie policy и добавить
  понятное уведомление.
- первая версия: агрегированная статистика из PostgreSQL через
  `deploy/scripts/report-stats.sh`, без browser cookies и сторонних трекеров.

## 3. Мониторинг и алерты

Статус: started.

- мониторить главную страницу, `/health`, очередь Redis, worker, Postgres,
  свободное место на диске и срок TLS-сертификата;
- добавить уведомления по email;
- отдельно отслеживать зависшие проверки и рост ошибок scanner worker.
- первая версия: `deploy/scripts/check-health.sh` и systemd timer на VPS.

## 4. Бэкапы и восстановление

Статус: started.

- включить ежедневный backup PostgreSQL;
- хранить несколько последних архивов вне рабочей директории проекта;
- регулярно проверять восстановление на тестовой базе;
- добавить короткую runbook-инструкцию на случай падения VPS.
- первая версия: `deploy/scripts/run-maintenance.sh` и systemd timer на VPS.

## 5. Защита от злоупотреблений

Статус: started.

- усилить rate limit по IP, по домену и по очереди;
- добавить admin denylist для доменов/IP, которые нельзя проверять;
- подготовить CAPTCHA после превышения порога;
- добавить firewall-level запрет исходящих запросов к приватным сетям из
  scanner containers.
- первая версия: `BLOCKED_CLIENT_IPS` и `BLOCKED_TARGETS` в `.env`.
- первая версия host firewall: `deploy/scripts/apply-egress-firewall.sh` и
  `tls-audit-egress-firewall.service`.

## 6. SEO-страницы под поисковый спрос

Статус: started.

- посадочная страница "Проверка SSL-сертификата онлайн";
- страница "Проверка TLS 1.2 и TLS 1.3";
- страница "Проверка HSTS";
- страница "Как получить оценку A+";
- страницы с готовыми конфигами для Nginx, Apache, Caddy/HAProxy;
- обновлять sitemap по мере появления страниц.
- первые страницы с A+, Nginx, Apache, Caddy и HAProxy добавлены.

## 7. Улучшение отчета

Статус: planned.

- сделать публичную ссылку на отчет понятной и копируемой;
- добавить сравнение "было/стало";
- добавить краткий executive summary для владельца сайта;
- оставить техническую глубину для администратора ниже по странице.

## 8. Экспорт и шаринг

Статус: planned.

- PDF-отчет;
- печатная версия страницы;
- JSON export с версией методики;
- кнопка "скопировать рекомендации".

## 9. Мониторинг сертификатов для пользователей

Статус: planned.

- подписка на проверку срока сертификата;
- email-уведомления до истечения сертификата;
- история проверок домена;
- без личного кабинета на первом этапе, через подтверждение владения доменом
  или одноразовую ссылку.

## 10. Публичный API и интеграции

Статус: deferred.

- API-ключи и лимиты;
- CI/CD check для релизов;
- webhook по завершению проверки;
- интеграции с популярными панелями/хостингами.
