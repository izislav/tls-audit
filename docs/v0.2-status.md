# TLS Audit v0.2 — Status Snapshot

Date: 2026-07-13
State: v0.2.1 deployed and verified in production

## 1) Current Product State

TLS Audit is in a working v0.2 stage:

- public scan flow works;
- monitoring subscriptions (free/pro) work;
- worker/scheduler/API run in production via Docker Compose;
- PostgreSQL is used for scan/report persistence;
- weekly security digest email flow is active;
- owner flow and private subscription management are active.

13. Monitoring admin surface protection:
    - trust-zone approach and restricted internal/admin endpoints.

14. Report UI fixes:
   - card stretch issue fixed;
   - section layout polished.

15. Security hardening in v0.2.1:
   - confirmation and management links expire;
   - confirmation URLs are no longer exposed by the public API;
   - HTTP ownership checks and scanner connections are pinned to validated public IP addresses;
   - public demo activation and account-status endpoints are removed;
   - email and monitoring mutations have dedicated rate limits;
   - production API documentation is disabled.

16. Reliability hardening in v0.2.1:
   - subscription-owned domains are scanned only by the subscription scheduler;
   - orphaned monitored domains are disabled automatically;
   - deleted subscriptions stop generating scans;
   - worker and scheduler heartbeats are part of readiness;
   - the public scan counter is lifetime-based and no longer decreases after retention cleanup;
   - stale unconfirmed subscriptions are removed by maintenance.

17. Runtime hardening in v0.2.1:
   - API, worker, and scheduler run as non-root users;
   - application filesystems are read-only with isolated temporary storage;
   - Linux capabilities are dropped and privilege escalation is disabled;
   - Docker logs are rotated and capped for every service;
   - system journal retention is capped at 300 MB and 14 days;
   - dependency audit reports no known vulnerabilities.

## 3) How It Works Right Now

1. User submits a domain for one-time scan and gets a report page.
2. User can enable monitoring subscription (free/pro) and confirm email.
3. Scheduler enqueues due scans, worker executes scans, API serves reports/status.
4. Scan results are stored and linked to monitoring snapshots/events.
5. Email digests are sent with cooldown and duplicate protection.
6. Pro flow respects ownership verification rules.
