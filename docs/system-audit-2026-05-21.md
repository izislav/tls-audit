# TLS Audit — System Audit Snapshot

Дата снимка: `2026-05-21`  
Статус: исторический снимок, не использовать как текущие оперативные метрики.

## Зачем нужен этот файл

Фиксирует контрольную точку состояния системы во время перехода к `v0.2`:

- локальная ветка опережала прод;
- прод имел схему БД старее текущей;
- потребовалась синхронизация `code + schema + docs`.

## Ключевой вывод снимка

Главный риск на тот момент был не в коде, а в дрейфе окружений:

1. локально доступны новые таблицы мониторинга/подписок;
2. на VPS часть новой схемы отсутствовала;
3. из-за этого часть функций мониторинга/Pro могла не работать корректно на проде.

## Что проверять при каждом новом аудите

1. `docker-compose ps` (health всех сервисов);
2. `curl /health`;
3. соответствие схемы БД текущему `deploy/postgres/schema.sql`;
4. размер БД, свободное место диска, размер backup-каталога;
5. прохождение тестов перед деплоем;
6. корректность SMTP-доставки и ссылок из писем;
7. отсутствие дублей weekly/alert отправок.

## Как использовать

- для актуальной проверки запускать новый live-аудит и сохранять отдельным файлом
  с текущей датой;
- этот файл оставлен как история и контекст для `v0.2`.
